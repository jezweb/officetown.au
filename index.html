Three prompt variants on the GitHub page: full install, cloud-only (if your agent host is already set up), connect-existing (wire a second machine to the same town).

Pro tip: .town is a real TLD (~$30/yr at Synergy, Hover, Porkbun). Register <yourbusiness>.town and point it at your deployment. Your agent already knows how to wire the DNS.

The shape

Four buildings. Four core roles. Add role packs as the work demands.

your-town/ ├── buildings/ │ ├── office/ # boss — routes work, holds the thread │ ├── library/ # librarian — extracts + curates the wiki │ ├── workshop/ # worker — deep work │ └── lookout/ # scout — outward scanning ├── wiki/ # the shared substrate │ ├── business/ # who you are │ ├── owner/ # voice + rhythm │ ├── team/ # humans + agents │ ├── contacts/ # external people │ ├── orgs/ # clients, prospects, vendors │ ├── projects/ # active + historical work │ ├── decisions/ # why we chose X │ ├── knowledge/ # graduated findings │ └── tasks/ # what's open, in flight, done └── AGENTS.md # the town's standing orders

The roles

Address them by name. Each knows what they do and what they don't.

💬 @boss

Talks to you. Routes work. Holds the thread across sessions. Never does deep work itself.

📚 @librarian

Extracts knowledge from your sources. Curates the wiki. Graduates findings into durable concepts.

⚙️ @worker

Deep work. Building. Research. Ships artefacts. Names trade-offs honestly.

🔭 @scout

Outward-facing. Industry. Tools. World. Surfaces what the inward roles wouldn't notice.
