Goose capabilities that work like a team

Office Town adds four addressable roles, a Cloudflare-backed wiki, and a catalogue of role packs to your Goose installation. Your single Goose chat becomes a team — boss routing work, librarian curating knowledge, worker building, scout scanning.

two steps

Click a button, paste a prompt, done

You have Goose installed already. Cloudflare provisions the backend in one click — D1, R2, Vectorize, Queue, Workers AI, Images, Email Routing — all auto-created from wrangler.jsonc. Then paste a short prompt into any capable AI agent to wire it into Goose. About 7 minutes end-to-end.

1. Deploy the backend

🚀 Deploy to Cloudflare

Cloudflare signs you in (no API token needed) and prompts for one required secret: MCP_BEARER_TOKEN. Generate via openssl rand -hex 32. ~2 minutes — hands you a worker URL.

2. Wire it into Goose

I want to add Office Town capabilities to my Goose installation. Office Town is a content bundle (Cloudflare Worker + Goose plugin + template) that installs INTO an existing Goose. NOT a Goose replacement. I'll provide: - My Cloudflare deployment URL (just got it from the Deploy button) - My MCP_BEARER_TOKEN (the one I entered in the Cloudflare deploy form) - A town folder path (default: ~/Documents/my-town) GROUND RULES: - Be transparent. Say what you're about to do. - Ask before destructive ops or any software install. - If Goose isn't installed: stop, point me at https://block.github.io/goose/. - Don't echo or save credentials. Env vars only. STEPS: 1. Check Goose is installed (`goose --version`). If not, stop. 2. curl -s /health — verify the deployment is reachable. 3. goose plugin install jezweb/office-town-plugin 4. goose plugin install jezweb/office-town-pack-knowledge 5. Edit ~/.config/goose/config.yaml — add 4 streamable_http extension entries (office-town-wiki / browser / devops / email) all pointing at /mcp/ with the bearer token in Authorization. Disable Goose's built-in `memory` extension in the same edit. 6. git clone https://github.com/jezweb/office-town . Seed wiki/knowledge/ from the office-town-pack-knowledge plugin. 7. Restart Goose. Smoke test: wiki.create a test entry, wiki.search finds it, @boss "introduce the team" works. Full prompt with verification checks: see INSTALL.md link below, or hit the Copy button — it copies the full ready-to-paste version.

The shape

Four buildings. Four core roles. Add role packs as the work demands.

your-town/ ├── buildings/ │ ├── office/ # boss — routes work, holds the thread │ ├── library/ # librarian — extracts + curates the wiki │ ├── workshop/ # worker — deep work │ └── lookout/ # scout — outward scanning ├── wiki/ # the shared substrate │ ├── business/ # who you are │ ├── owner/ # voice + rhythm │ ├── team/ # humans + agents │ ├── contacts/ # external people │ ├── orgs/ # clients, prospects, vendors │ ├── projects/ # active + historical work │ ├── decisions/ # why we chose X │ ├── knowledge/ # graduated findings │ └── tasks/ # what's open, in flight, done └── AGENTS.md # the town's standing orders

The roles

Address them by name. Each knows what they do and what they don't.

💬 @boss

Talks to you. Routes work. Holds the thread across sessions. Never does deep work itself.

📚 @librarian

Extracts knowledge from your sources. Curates the wiki. Graduates findings into durable concepts.

⚙️ @worker

Deep work. Building. Research. Ships artefacts. Names trade-offs honestly.

🔭 @scout

Outward-facing. Industry. Tools. World. Surfaces what the inward roles wouldn't notice.

Get started

Three steps, ~7 minutes total. The button does the heavy lifting; your AI agent wires it into Goose.

StepWhatTime
1.Install Goose if you don't have it (Desktop or CLI, either works)~2 min
2.Click Deploy to Cloudflare → sign in, paste a bearer token, click Deploy. Cloudflare provisions D1, R2, Vectorize, Queue, Workers AI, Browser Rendering. Hands you a worker URL.~2 min
3.Paste the prompt from INSTALL.md into a capable AI agent → it installs the Goose plugin, wires the four MCP servers to your worker URL, clones the town template, runs a smoke test.~3 min

The agent doing step 3 can be Goose itself, Claude Code, Aider, Cline — anything that can run shell commands and edit YAML. It never touches wrangler or your local toolchain; Cloudflare's web flow handled all of that in step 2.
